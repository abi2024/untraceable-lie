## How to write an entry (copy this shape)
```
### <ISO timestamp> · <Tn — task name> · <PASS | FAIL | GATE | NOTE>
- What I did: <1–3 lines, concrete>
- Outputs produced: <exact file paths>
- Done-check result: <the numbers/booleans, e.g. "C1=2.3GB PASS, C3=19 GPU-hr / $114 PASS">
- Guards run: <which guards, and their result>
- Decision / next: <what this routes to; any gate colour; anything surfaced to human>
- Cost so far: <cumulative GPU-hr and $ if applicable>
```

**Entry types:**
- `PASS` — task's done-check passed; record the verifying numbers.
- `FAIL` — done-check or a guard failed; record the exact error and what you'll try. Set `03_STATE.md` to BLOCKED.
- `GATE` — a gate fired; record the inputs, the rule applied, the colour/branch chosen, and whether human confirmed.
- `NOTE` — an assumption made under ambiguity, a surfaced decision, or a deviation worth recording.

**Rules:**
1. Every number that ends up in a result must trace to a PASS entry here that names its artifact.
2. Any locked-decision conflict (from `01_PRD`) is logged as a `NOTE` and surfaced to the human — never silently resolved.
3. Cumulative cost is updated on every entry that used GPU time, so budget burn is always visible.

  - UQC: score = logprob(yes-synonyms) − logprob(no-synonyms) at first response token. yes/no confirmed CLEAN single tokens (yes=13059, no=2649, + case/space variants) → first-token scoring valid and Pacchiardi-faithful.

UQC stays first-token (clean tokens, Pacchiardi-faithful); only Self-Eval switched to full-word — method matched to tokenization, verified against the actual tokenizer on Windows.
